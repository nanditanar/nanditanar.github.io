Introduction

Our project, "GreenPlate," is a sustainable food management system designed to effectively promote sustainable eating and shopping habits. By minimizing food waste and encouraging the efficient use of resources, GreenPlate aims to help users adopt a more sustainable lifestyle. The application provides functionality for users to track their daily calorie intake, manage recipes, monitor ingredient quantities, and generate optimized shopping lists. Through these features, GreenPlate allows users to make informed decisions about their food consumption and shopping, contributing positively to environmental sustainability. In an era where the world wastes over 2.5 billion tons of food a year, sustainability is crucial, and managing food resources efficiently is imperative. GreenPlate addresses this by integrating tools that assist users in planning their meals and shopping trips to reduce waste and maximize the use of available resources. By promoting practices that support sustainability, GreenPlate plays a pivotal role in fostering responsible consumer habits, which are essential for the long-term health of our planet.

Design & Architecture

The architecture of GreenPlate is built on the Model-View-ViewModel (MVVM) pattern, which facilitates a clear separation of concerns and enhances the maintainability and scalability of the application. This architectural choice is ideal for handling the dynamic features provided by GreenPlate. Several design patterns are employed to ensure the robustness and flexibility of the application:

  • Singleton: Used for managing user sessions, ensuring that only one user instance is active at any given time.
  • Strategy: Allows for flexible sorting mechanisms in recipe management, where different sorting strategies can be applied dynamically.
  • Factory: Used for creating instances of various models and views, simplifying the management of these components.

UML Diagrams

  • This Design Class Diagram (DCD) provides an overview of the classes and relationships in the "GreenPlate" app.
    1. User Class: Central to the application, it includes attributes for username, password, height, weight, and gender. It's implemented as a Singleton, meaning only one instance exists at any given time, ensuring consistency across the application.
    2. LoginViewModel: Manages the login functionality with methods like `onLoginClicked()` and properties such as `user`, `successMessage`, and `errorMessage`. It validates input data and interacts with the `User` class.
    3. AccountCreationViewModel: Handles new account creation, validating input data and creating new user accounts. It utilizes the `User` class for account information.
    4. MealInfo Class: Represents the details of a meal, including name, calories, and date.
    5. InputMealViewModel: Processes meal input, verifying the validity of the data entered against the `MealInfo` class.
    6. IngredientsModel: Contains information about individual ingredients such as name, quantity, calories, and expiration date.
    7. IngredientsViewModel: Manages ingredient-related operations, like checking for ingredient existence, with a callback interface `IngredientCheckCallback` for response handling.
    8. RecipeModel: Represents a recipe with attributes like name, a map of ingredients and their quantities, and a flag indicating if there are enough ingredients for the recipe.
    9. RecipeViewModel: Manages recipe operations and interacts with the `RecipeModel`. It uses the Firebase database for storage and retrieval of recipe data.
    10. `db` (FirebaseDatabase instance) and `root` (DatabaseReference) are initialized using the Singleton pattern in `IngredientsViewModel` and `RecipeViewModel`. This ensures a single reference point for database operations related to ingredients and recipes.

User Interface

GreenPlate's user interface is designed with intuitiveness and ease of use in mind. The UI employs a simple yet effective layout that guides users through the functionalities seamlessly.

  • Meal Logging: Users can log their meals, including details such as calorie count and ingredients used.
  • Recipe Management: Allows users to create, store, and manage recipes with an integrated ingredient inventory system.
  • Shopping List Generation: Automatically populates shopping lists based on the ingredients needed for upcoming meals.
  • Welcome Screen: The first screen users see when they open the app, providing options to either start using the app or exit.
  • Meal Input Screen: Where users can enter meal details for tracking.
  • Recipe Screen: Displays stored recipes and allows for the addition of new ones.
  • Shopping List Screen: Shows the ingredients needed for the selected recipes, highlighting what needs to be purchased.

These screenshots not only demonstrate our app’s features but also highlight the user-friendly design and logical flow between different functionalities.

Conclusions and Reflections/Learning
